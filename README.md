## Задание №4: Приложение для работы с таблицей падежей существительных

### Описание проекта
Данное приложение предназначено для обработки существительных и определения их начальной формы (именительного падежа) и текущего падежа на основе предварительно загруженной базы данных (таблицы склонений).

### Реализованные требования:
1.  **STL**: Ядро приложения построено на использовании контейнеров (`std::vector`, `std::map`, `std::unordered_map`) и алгоритмов (`std::find_if`, `std::transform`, `std::for_each`).
2.  **C++ Style**: В коде отсутствуют С-конструкции. Используются потоки ввода-вывода (`fstream`, `sstream`), строки `std::string`, а вместо `printf` и указателей — современные средства C++.
3.  **Интерфейс**: Реализован удобный графический интерфейс на базе **Qt Framework** (согласно условию выбора библиотеки).
4.  **Логирование**: Система логирует действия пользователя, успешные операции и ошибки в файл `application.log` с указанием времени, важности (INFO, WARNING, ERROR) и текста сообщения.
5.  **Безопасность**: Реализована «защита от дурака», валидация входных данных и обработка исключений (`try-catch`, `std::runtime_error`).

---

### Структура входных данных

#### 1. База данных (Словарь склонений)
Для корректной работы необходимо загрузить файл словаря в следующем формате:
```text
Кот
Именительный: кот
Родительный: кота
Дательный: коту
Винительный: кота
Творительный: котом
Предложный: коте
```
*Важно: После названия слова должны идти ровно 6 строк с падежами через двоеточие.*

#### 2. Обрабатываемый файл
Файл со списком слов, которые нужно распознать. Каждое слово на новой строке:
```text
коту
кота
окном
```

---

### Функционал приложения
*   **Загрузка базы**: Кнопка «Загрузить файл» позволяет импортировать таблицу склонений. Данные отображаются в интерактивной таблице в главном окне.
*   **Поиск слова**: Позволяет мгновенно проверить одно слово через строку поиска.
*   **Пакетная обработка**:
    1. Выберите входной файл со словами.
    2. Укажите путь для сохранения результата.
    3. Нажмите «Обработать».
*   **Просмотр логов**: В нижней части окна отображается история всех событий в реальном времени.

---

### Сборка и запуск
1.  **Среда**: Qt Creator или Visual Studio с расширением Qt.
2.  **Стандарт**: C++17 или выше.
3.  **Зависимости**: Библиотеки Qt5 или Qt6 (Core, Gui, Widgets).
4.  **Файл логов**: При запуске приложение создает `application.log` в рабочей директории.

---

### Технические особенности реализации
*   **`DeclensionDatabase`**: Использует `std::unordered_map` для «обратного поиска» (reverse lookup), что обеспечивает поиск любого падежа слова за константное время $O(1)$.
*   **`FileProcessor`**: Использует лямбда-выражения и STL-алгоритмы для эффективной обработки файлов больших объемов.
*   **`Logger`**: Реализован по паттерну Singleton, является потокобезопасным (использует `std::mutex`).
*   **Интерфейс**: Динамически блокирует кнопки управления, если база данных еще не загружена (защита от некорректных действий).
